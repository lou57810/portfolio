Utilisez les bases de Python pour l'analyse de Marché

Objectifs du projet


Vous êtes analyste marketing chez Books Online, une importante librairie en ligne spécialisée dans les livres d'occasion. Dans le cadre de vos fonctions, vous essayez de suivre manuellement les prix des livres d'occasion sur les sites web de vos concurrents. Cela représente trop de travail et vous n'arrivez pas à y faire face : il y a trop de livres et trop de librairies en ligne ! Vous et votre équipe avez décidé d'automatiser cette tâche laborieuse via un programme (un scraper) développé en Python, capable d'extraire les informations tarifaires d'autres librairies en ligne.

L'objectif est de développer une version bêta de ce système pour suivre les prix des livres chez Books to Scrape, un revendeur de livres en ligne.

Gerez du code avec Git et Github

Git est un gestionnaire de versions

C'est un programme qui permet aux développeurs de conserver un historique des modifications et des versions de tous leurs fichiers.
Cet outil a donc trois grandes fonctionnalités :
- Revenir à une version précédente de votre code en cas de problème.
- Suivre l’évolution de votre code étape par étape.
- Travailler à plusieurs sans risquer de supprimer les modifications des autres collaborateurs.

Dépot distant

Création d'un compte sur Git Hub et paramètrages sécurités, et ajouter le new repository.

Dépot local

Télécharger et installer Git en local.
Puis configurer Git:

Principe de fonctionnement:

Le Working directory
Cette zone correspond au dossier du projet sur votre ordinateur.

Le Stage ou index
Cette zone est un intermédiaire entre le working directory et le repository.
Elle représente tous les fichiers modifiés que vous souhaitez voir apparaître dans votre prochaine version de code.

Le Repository
Lorsque l’on crée de nouvelles versions d’un projet, elles sont stockées dans le repository.
Avant de faire un push sur Git hub, il est indispensable de créer un fichier '.gitignore'
dans lequel nous ajouterons les fichiers que nous ne voulons pas envoyer.
Puis créer le lien avec Git Hub après avoir initialisé les clés (SSH, HTTPS, GPG)
En remplaçant votre nom Git et nom Projet.




Enfin pusher les fichiers en 3 étapes:
- git add .
- git commit -m "commentaire"
- git push
Le commentaire est important pour retrouver facilement l'évolution des dépots.

L'intérêt de Git est également de créer plusieurs branches'

- git branch nouvelle branche
- git checkout nouvelle branche(ex: develop)
Le code peut continuer à évoluer dans la nouvelle branche, puis être 'mergé' dans une autre branche
Ex: 'git checkout branche main'
'git merge develop'

Il est également possible de rapatrier et travailler sur un projet Git avec la commande fork.
Le développment local peut alors progresser et mis à soumissions sur le projet original avec
une 'pull request' effectué depuis Git hub.

Bien d'autres commandes peuvent aussi être réalisées avec Git comme des corrections en local ou distant.
