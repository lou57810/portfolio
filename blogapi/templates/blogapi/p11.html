Projet 11

Objectif:

En tant que développeur Python dans ce projet, vous utiliserez des outils comme : Sentry pour surveiller l'application et détecter rapidement les erreurs ; Read The Docs pour documenter l'utilisation de l'application ; et Github, Github compose, GitHub Actions ou GitLab pour les pipelines CI/CD qui automatisent les tests et le déploiement. vous allez ajouter une démarche essentielle : la factorisation d’une application existante pour améliorer sa maintenabilité, sa performance et sa scalabilité. Vous identifierez et corrigerez les points faibles du code pour en améliorer la performance.

Dans ce projet, mon mentor préconise l'utilisation de github comme VCS(Version Control System), conjointement avec docker. Celà nécéssite la création d'un fichier Dockefile.

Ensuite le principe est d'utiliser une plateforme d'intégration continue pour mettre en oeuvre les pratiques Devops.

Pour ce projet, comme indiqué dans les requis nous utilisons CircleCi, plateforme CI/CD d'intégration continue et de livraison continue.

CircleCI surveille les référentiels GitHub, et d'autres, et lance des builds pour chaque nouveau commit.

CircleCi teste automatiquement les builds dans les conteneurs Docker ou les machines virtuelles, et déploie les builds de passage vers les environnements cibles. Un tableau de bord et une API permettent de suivre l'état des builds et les métriques liées aux builds. Une intégration Slack informe les équipes en cas de problèmes.

La prise en charge de SSH permet d'exécuter des tâches localement, et des mesures de sécurité empêchent la falsification. CircleCi propose également une fonction d'approbation du flux de travail, qui met en pause le(s) jusqu'à ce qu'une approbation manuelle soit donnée.

Orbs

Les orbes sont des extraits partageables de YAML, qui peuvent être utilisés pour simplifier les builds CircleCi, et éffectuer des déploiements.

CircleCi nécéssite la création d'un fichier config.yml

Mise en place Docker

Un conteneur Linux est un processus ou un ensemble de processus isolés du reste du système, tout en étants légers.

Il ne crée qu'une isolation des processus, et partage les ressources avec le système hôte.
En pratique le processus n'utilisera que la RAM nécéssaire laissant le reste disponible pour d'autres processus.
Les conteneurs étant légers et rapides au démarrage, il permettent de créer des environnements dynamiques et ainsi répondre à des besoins métiers.

Docker a été créé pour répondre à des besoins de PaaS (Platform as a Service)

Docker Hub est un service fourni par Docker Inc ; vous pouvez le comparer à GitHub, mais spécialisé dans le stockage d'image pour Docker.
