Python: Programmation Orientée Objet

Les classes:

Les classes sont des conteneurs,à l'intérieur desquels nous pouvons définir des attributs,
ainsi que des méthodes pouvant modifier ces attributs.
Ces classes permettent alors de créer des objets, possédant ces attributs et ces méthodes.
Il est alors possible d'instancier plusieurs objets ayant leurs attributs propres,
et des méthodes à leur disposition exclusives.

Pour résumer de façon imagée, une classe est une sorte de modèle ou recette permettant de créer un objet,
qui pourrai être un personnage, un véhicule, ou un plat cuisiné.

La POO complémente la programmation procédurale en y ajoutant une couche d'abstraction.

L'exercice proposé dans ce projet consiste à:
- Créer des classes qui nous serviront de modèles pour le tournoi, les joueurs, les matchs et les rondes.
- Utiliser des views pour afficher les classements et autres appariements.
- Implémenter des contrôleurs pour accepter les données de l'utilisateur,
produire les résultats des matchs, lancer de nouveaux tournois, etc.

Code Python conforme à la PEP8

Commenter le code pour une meilleure compréhension en relecture pour un autre programmeur ou soi-même.
Les docstrings permettent de bien documenter le code.
La PEP8 introduit des conditions de nommage qui permettent de distinguer les éléments du code
comme les variables, les classes, les fonctions, les modules, etc....
Flake8 signalera également une mauvaise indentation, les 'blank lines', les espaces, la longueur des lignes.

Respecter le concept 'SOLID'
- S désigne la responsabilité unique (Single responsibility : Chaque classe ou fonction
doit faire une seule chose, et la faire bien).
- O désigne le principe ouvert/fermé (Open/Closed: Les classes doivent être ouvertes à l’extension,
mais fermées à la modification).
- L désigne la substitution de Liskov (Les sous-classes doivent pouvoir faire tout ce que font
leurs classes parentes).
- I désigne la ségrégation des interfaces (Interfaces Segregation: Cela correspond essentiellement
au principe de responsabilité unique, appliqué aux interfaces.)
- D désigne l'inversion des dépendances (Dependance Inversion: Les classes parentes
ne doivent pas avoir à changer lorsque l’une de leurs sous-classes est modifiée).

Utiliser un linter

En console avec la commande 'flake8' ou en version page html,
'flake8 --format=html --htmldir=flake-report', flake8 permet la vérification
de la conformité du code à la PEP8
