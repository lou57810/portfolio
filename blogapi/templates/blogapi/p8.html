Django Rest Framework est une surcouche à Django et permet entre autres d’automatiser la gestion CRUD
(Create, Read, Update, Delete) des modèles sous forme d’API.

API: Application Programming Interface




L'api est un moyen de communication entre deux logiciels locaux ou distants,
que ce soit entre composants d'une application ou entre deux applications différentes:
transmettre, partager et échanger des informations, de manière fluide et uniforme,
en collaborant pour accomplir des tâches spécifiques.

Les API permettent de centraliser toute l’information en ayant moins de données et de logique métier
à gérer dans les terminaux. Une seule application Backend proposant une API
peut être mutualisée et être consommée par une application Front,
des applications mobiles Android/iOS, mais aussi d’autres systèmes d’information, des partenaires, etc.

Une api permet la communication entre une app client et une app serveur: elle sert d'intermédiaire:


Api privées: elle permet uniquement aux utilisateurs autorisés de l'application d'utiliser l’API
qui peut accéder à la base de données.

Api publique: Disponibles à tous

Il est possible de restreindre l'accès à une Api à des utilisateurs authentifiés ou bénéficiant d'un niveau
d'accès sélectif.

DRF: Django Rest Framework

REST:
- Client-serveur séparation:
- Le client: application, logiciel ou navigateur, formule une requête, via l'Api
- Serveur: Il récupère et transmet les réponses depuis la base de données.
- Stateless: Le serveur ne sauvegarde aucune des requêtes ou réponses précédentes.
- Le code devra être précis, et la réponse déterminée et compréhensible.
- Cacheable
- La réponse peut être mise en cache et renvoyer la même réponse avec son n° de version si la requête est identique.
- Uniforme Interface
- Chaque développeur doit utiliser les mêmes normes. Les données doivent être identifiées par un nom ou une adresse web (uri: uniforme ressource identifier)
- Layered system
- Chaque composant qui utilise REST n’a pas accès aux composants au-delà du composant précis avec lequel il interagit.
- Code on demand
- Le code à la demande signifie que le serveur peut étendre sa fonctionnalité en envoyant le code au client pour téléchargement.
