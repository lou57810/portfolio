API

Un moyen de communication entre deux logiciels, que ce soit entre différents composants d’une application
ou entre deux applications différentes. C’est une abstraction :
Une façade qui simplifie des mécanismes complexes permettant de fournir un service demandé.

Un service web permet de manipuler les données qu’il expose à travers son API en ligne.
Il peut les créer, les récupérer, les mettre à jour et les supprimer.
Réspectivement: 'POST', 'GET', 'PUT', 'DELETE'. Ces quatre opérations permettent de persister vos données, c’est-à-dire de les stocker durablement.
Ainsi, lorsque une application web communique avec le service web,
ces manipulations sont sauvegardées jusqu’à la prochaine utilisation.

On parle d’API HTTP lorsqu’un service web offre une façade à travers le protocole HTTP.
Le serveur web munis de son url, est un programme informatique qui met les services web
à disposition des autres programmes.
Le navigateur et le serveur doivent donc communiquer entre eux grâce au protocole HTTP.
Le navigateur web utilise le protocole HTTP pour envoyer des requêtes à destination du serveur.
