Empêcher les injections SQL:

L'injection SQL compte parmi les attaques les plus redoutables et figure au top 10 de l'OWASP(N°3) (Open Web Application Security Project).

Le but est de sécuriser et rendre une application crédible.

1. Usage de la dernière version de sqlalchemy:

Sqlalchemy (Depuis 2006) est un toolkit opensource SQL et dispose d'une librairie ORM (Object Relational Mapping) écrite en Python.

Sqlalchemy utilise le pattern DataMapper: Les SQL utilisent différents mechanismes entre les modèles Objets, les logiques de fonctionnement(business logics) et les bases de données.

Les schemas objets, et les schemas relationnels ne correspondent pas. Les objets en mémoire, entrent en collision avec la structure de la base de donnée.

DataMapper constitue une couche programme, qui sépare les objets en mémoire de la base de donnée. Sa responsabilité est de les isoler l'une de l'autre, tout en transférant les données entre elles.

Les objets en mémoire n'ont pas connaissance d'une éventuelle présence de base de donnée. Ils n'ont pas besoin d'un code d'interface SQL et encore moins connaissance du schema de la base de données.

Réciproquement, la base de données n'a aucune idée des objets qui l'utilisent.Le mapping s'éffectue de façon invisible.DataMapper est lui-même inconnu de la couche Domain.

2. Aucun nom d'utilisateur ne doit figurer dans le programme, d'ou l'implémentation du fichier .env qui doit être similaire à env_template et contenir toutes les valeurs à renseigner.

3. Utilisation de MySQL, qui est une SGBD sécurisée.

4. Les collaborateurs sont dirigés vers leur menus respectifs.

5. De surcroit, les permissions sont systématiquements vérifiées en fonction du role collaborateur.
